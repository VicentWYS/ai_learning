## 初始条件
1. 一台全新的电脑，没有安装任何环境；
2. 在本地新建了一个文件夹。


## 安装环境
### 安装Git
1. 进入这个链接，根据指示安装即可：[Git](https://git-scm.com/?utm_source=chatgpt.com)
2. 验证：打开 powershell, 输入 `git --version`,若能看到版本号，说明安装成功。

### 安装 python
推荐：直接让 uv 管理 Python，后面 uv 会自动下载并管理，这是现在很多 AI 工程项目的做法。

### 安装 uv
1. 打开 powershell，输入： 

```shell
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
```

2. 验证：powershell 中输入：`uv --version`,如果出现版本号，说明安装成功。


## 初始化环境
### 创建Github仓库
1. 在Github上创建一个新的仓库，不要勾选`README`

### 初始化 Git 仓库
1. 进入文件夹，用 vscode 打开；
2. 新建终端，输入：`git init`，这时，该项目已经开始被 Git 管理了。

### 关联Github远程仓库
1. 复制仓库地址（如：`
https://github.com/你的用户名/my-project.git`）
2. 在终端中输入：

```
git branch -M main
```

```
git remote add origin 仓库地址
```

验证：输入 `git remote -v`，如果能看到远程仓库地址，说明关联成功。

### 初始化 python 项目
1. 终端中输入：`uv init`，它会自动生成：

```
pyproject.toml
main.py
README.md
```

### 用 uv 自动安装 python
1. 终端中输入：

```
uv python install 3.11
```

2. 验证：`uv python list`，会看到诸如：`cpython-3.12.x`

### 创建项目独立虚拟环境
1. 终端中输入：

```
uv venv
```

2. 这意味着该项目拥有了： 独立 Python、独立 pip、独立依赖、不污染系统环境。

### 激活虚拟环境
1. 终端中输入：

```
.venv\Scripts\activate
```

1. 验证：命令行前面会出现 `(.venv)` 或 本项目的名称；


## 安装依赖
### 安装 langchain
1. 终端中依次输入并执行以下指令：

```
uv add langchain
uv add langgraph
uv add openai
uv add langchain-openai
uv add python-dotenv
```

然后它会自动安装依赖、更新 pyproject.toml、生成 uv.lock；


## 项目管理文件
### 理解 pyproject.toml
1. 打开 `pyproject.toml`，可看到项目的全部依赖，后续整个项目依赖体系，都在这里管理。

```yaml
[project]
name = "langchain-learning-uv"
version = "0.1.0"
description = "Add your description here"
readme = "README.md"
requires-python = ">=3.11"
dependencies = [
    "langgraph>=1.2.0",
    "openai>=2.37.0",
    "python-dotenv>=1.2.2",
]
```

### 理解 uv.lock
项目根目录中的文件 `uv.lock` 是： “依赖快照”。 它会精确记录项目里每一个包的确切版本。
这样，无论换电脑还是部署服务器： `uv sync` 就能恢复完全一致的环境。这就是现代工程化。


## 配置 .gitignore
1. 创建：`.gitignore`
2. 写入：

```bash
# VS Code
.vscode/

# JetBrains IDE（PyCharm / IDEA）
.idea/

# Python 缓存
__pycache__/
*.pyc
*.pyo

# 虚拟环境
venv/
.env/
.venv/

# 日志
*.log
logs/

# 本地配置
.env
*.env
config.local.*

# 数据库文件
*.sqlite

# 思维导图文件
*.drawio/
```

注意：虚拟环境不能提交、缓存文件不能提交、API Key 不能提交。


## 设置依赖文件
1. 创建 `.env` 文件，写入模型相关的**url**和**key**
2. 创建 `.env.example` 文件，复制 `.env` 的内容，将**key**等敏感信息替换为说明性的占位符
