# LangChain 结构化输出（OutputParser）知识点

## 一、什么是结构化输出

大模型默认返回的是**自然语言文本**，不利于程序后续处理。**结构化输出**指让模型按预定格式（如 JSON、Pydantic 模型）返回结果，便于解析和类型安全使用。

## 二、核心概念

### 1. Pydantic 模型（Schema）

用 **Pydantic** 的 `BaseModel` 定义期望的输出结构，每个字段用 `Field(description=...)` 描述，便于模型理解要生成什么：

```python
from pydantic import BaseModel, Field

class Person(BaseModel):
    """人物信息"""
    name: str = Field(description="姓名")
    age: int = Field(description="年龄")
    occupation: str = Field(description="职业")
```

### 2. PydanticOutputParser

- **作用**：把「Pydantic 模型」和「与模型的对话」桥接起来。
- **两件事**：
  1. **生成格式说明**：`parser.get_format_instructions()` 会得到一段告诉模型「请按如下 JSON Schema 输出」的文本。
  2. **解析模型输出**：`parser.parse(raw_content)` 把模型返回的字符串（通常是 JSON）解析成 Pydantic 对象。

```python
from langchain_core.output_parsers import PydanticOutputParser

parser = PydanticOutputParser(pydantic_object=Person)
format_instructions = parser.get_format_instructions()  # 用于拼进 prompt
# ...
person_obj = parser.parse(raw_content)  # 将模型输出解析为 Person 实例
```

### 3. 与 Prompt 的结合

把 `format_instructions` 拼进用户或系统提示中，模型才会按约定格式输出：

```python
user_prompt_full = f"{user_prompt}\n\n{format_instructions}"
```

这样模型会看到「按某某 JSON Schema 输出」的说明，返回的文本才容易被 `parser.parse()` 正确解析。

## 三、整体流程简述

1. **定义** Pydantic 模型（如 `Person`）。
2. **创建** `PydanticOutputParser(pydantic_object=Person)`。
3. **组装 Prompt**：`user_prompt` + `parser.get_format_instructions()`。
4. **调用** `model.invoke(messages)`，得到 `response.content`（原始字符串）。
5. **解析**：`parser.parse(raw_content)` → 得到 `Person` 等结构化对象。
6. **使用**：通过 `person_obj.name`、`person_obj.model_dump()` 等在代码中使用。

## 四、运行代码实例

本目录下 **`main.py`** 中包含两个可运行示例：

| 示例 | 函数 | 说明 |
|------|------|------|
| 对照组 | `example_1_without_outputparser()` | 不使用 OutputParser，模型自由文本回复 |
| 结构化输出 | `example_2_outputparser()` | 使用 PydanticOutputParser，输出解析为 Person 对象 |

运行方式：

```bash
cd "phase2_practical/11_structured_output/02 - outputparser 结构化输出"
python main.py
```

需在项目根目录的 `.env` 中配置 `QWEN_API_KEY` 和 `QWEN_BASE_URL`。
