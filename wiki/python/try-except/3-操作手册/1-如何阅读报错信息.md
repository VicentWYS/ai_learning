# 如何阅读 Python 报错信息 —— 从崩溃到精准捕获

> 面对一屏红色报错不慌张，三眼就能找到该 catch 什么。

---

## 一、先看一个真实报错长什么样

假设你写了这样一段裸代码（没有 try-except）：

```python
from langchain.chat_models import init_chat_model

model = init_chat_model(
    model="deepseek-v4-flash",
    model_provider="openai",
    api_key="sk-123456",        # ← 故意写一个假 key
    base_url="https://api.deepseek.com/v1",
)

response = model.invoke("你好")
print(response.content)
```

运行时你会看到类似这样的报错（不同 SDK 版本输出略有差异）：

```
Traceback (most recent call last):
  File "D:\my_project\app.py", line 12, in <module>
    response = model.invoke("你好")
  File "D:\my_project\.venv\lib\site-packages\langchain\chat_models\base.py", line 256, in invoke
    return self._generate(messages, stop=stop, ...)
  File "D:\my_project\.venv\lib\site-packages\langchain\chat_models\openai.py", line 345, in _generate
    response = self.client.chat.completions.create(...)
  File "D:\my_project\.venv\lib\site-packages\openai\resources\chat\completions.py", line 234, in create
    return self._post(...)
openai.AuthenticationError: Error code: 401 - {'error': {'message': 'Incorrect API key provided', ...}}
```

## 二、三眼法：逐层解读报错

把报错从上到下分成三层来看：

```
                        👁 第一眼看这里
Traceback (most recent call last):
                        ↓
  File "...app.py", line 12, in <module>      ← 你的哪行代码触发的
    response = model.invoke("你好")
                        ↓
  File "...langchain\chat_models\base.py"...  ← 库内部调用链（通常可以跳过）
    ...
  File "...openai\resources\chat\..."...      ← 库内部调用链
    ...
                        👁 第二眼看这里（最后一行）↓
openai.AuthenticationError: Error code: 401 - ...  ← 异常类型 + 错误原因
```

### 第一眼：看最后一行 —— 决定 catch 什么

```
openai.AuthenticationError: Error code: 401 - {'error': {'message': 'Incorrect API key...}}
│                         │
│                         └── 具体原因：API Key 不对
└── 异常类型 ← 这就是你要写在 except 后面的名字
```

**拿到异常类型后，翻到代码里对应的位置加处理：**

```python
try:
    response = model.invoke("你好")
except openai.AuthenticationError as e:   # ← 直接从最后一行抄过来
    print(f"API Key 无效：{e}")
```

### 第二眼：看倒数第一个你自己的文件 —— 决定在哪加 try

从下往上扫，找到**第一个属于你自己项目的文件**：

```
  File "D:\my_project\app.py", line 12, in <module>   ← 就是这里
    response = model.invoke("你好")
```

`line 12` 就是你需要包上 `try-except` 的那一行。

### 第三眼：看异常消息 —— 决定给用户什么提示

```
Error code: 401 - Incorrect API key provided
```

消息里的关键词告诉你怎么写提示语：
- `Incorrect API key` → "请检查 API Key 是否正确"
- `Connection refused` → "网络连接失败，请检查网络或 Base URL"
- `Rate limit` → "调用太频繁，稍后重试"

## 三、AI 开发中最常见的几种报错及应对

### 1. 认证 / 权限类

**报错示例：**
```
openai.AuthenticationError: Error code: 401 - ...
```

**应对：**
```python
except openai.AuthenticationError as e:
    print(f"🔑 API Key 无效或已过期，请检查 .env 文件：{e}")
```

### 2. 网络 / 连接类

**报错示例：**
```
ConnectionError: [Errno 10061] Connection refused
# 或者
requests.exceptions.ConnectionError: ... Max retries exceeded ...
# 或者
urllib3.exceptions.NewConnectionError: ...
```

**应对：**
```python
except (ConnectionError, requests.exceptions.ConnectionError) as e:
    print(f"🌐 无法连接到 API 服务器，请检查网络和 Base URL：{e}")
```

### 3. 超时类

**报错示例：**
```
TimeoutError: Request timed out after 60 seconds
# 或者
openai.APITimeoutError: Request timed out.
```

**应对：**
```python
except TimeoutError as e:
    print(f"⏰ API 响应超时，正在重试...")
    # 可以考虑在这里加重试逻辑
```

### 4. 参数 / 值错误

**报错示例：**
```
ValueError: Invalid model name 'gpt-xxx'
# 或者
TypeError: invoke() got an unexpected keyword argument 'xxx'
```

**应对：**
```python
except ValueError as e:
    print(f"⚙️ 参数配置有误：{e}")
except TypeError as e:
    print(f"⚙️ 调用方式有问题：{e}")
```

### 5. SDK 自定义异常（需要看文档）

许多 LLM SDK 有自己的异常体系，比如 OpenAI SDK：

```python
import openai

try:
    response = model.invoke("你好")
except openai.APIError as e:
    # 所有 OpenAI 异常的基类
    print(f"API 错误：{e}")
except openai.RateLimitError as e:
    # 调太快了
    print(f"频率受限，等一会儿再试：{e}")
```

## 四、实战演练：根据报错写出正确的异常处理

### 题目

你在运行下面的代码时收到了报错，请写出正确的异常处理：

```python
# 现状：裸代码
response = model.invoke("用 Python 写一个快速排序")
```

**报错信息：**
```
Traceback (most recent call last):
  File "D:\project\main.py", line 25, in generate_code
    response = model.invoke("用 Python 写一个快速排序")
  File "...\langchain\chat_models\openai.py", line 345, in _generate
    ...
openai.RateLimitError: Error code: 429 - {'error': {'message': 'You exceeded your current quota, please check your plan and billing details.', 'type': 'insufficient_quota'}}
```

### 你的分析过程

**第一眼（最后一行）：** 异常类型是 `openai.RateLimitError`
**第二眼（你的文件）：** `main.py` 第 25 行
**第三眼（消息）：** `insufficient_quota` —— 额度用完，不是调太快

### 写出的异常处理

```python
try:
    response = model.invoke("用 Python 写一个快速排序")
except openai.RateLimitError as e:
    # 再区分一下：是临时限流还是额度真用完了
    if "insufficient_quota" in str(e):
        print("💸 API 额度已用完，请充值或更换 API Key")
    else:
        print("⏳ 调用太频繁，等待 10 秒后重试...")
        import time
        time.sleep(10)
```

## 五、总结：从报错到代码的流程

```
运行代码 → 看到红色报错
              │
              ▼
     1. 看最后一行 → 拿到异常类型名
              │
              ▼
     2. 从下往上找 → 找到你自己的文件 + 行号
              │
              ▼
     3. 读报错消息 → 理解错误原因
              │
              ▼
     4. 在该行外包 try-except，用第1步的类型
              │
              ▼
     5. 再次运行 → 验证是否被正确捕获
```

**记住一句话：报错不是敌人，是免费测试报告。** 每次崩溃都在告诉你："这里有坑，应该这样处理。"你要做的就是把最后一行异常类型抄到 `except` 后面。
